Explore the journey behind who we are

The journey of our project began with a detailed planning session, where we laid out the project’s goals, assigned tasks, and decided on the tools and technologies we would use. The primary objective was to create an interactive website, which would involve not only technical development but also collaboration and coordination among team members. Throughout this process, we made use of several tools for design, development, and collaboration, which helped shape this project. This report summarizes our approach, the challenges faced, and the lessons learned during the course of the project.

Design and planning phase

We started by focusing on the design of the website, as this would lay the foundation for the development phase. Figma, a user interface design tool, was our primary tool for creating wireframes and visual designs of the website pages. Using Figma allowed us to experiment with different layout, user flow, and interactions, which ultimately helped in refining the overall look and feel of the website. In parallel, we used Coolers, an online color scheme generator, to ensure the design adhered to an aesthetically pleasing and consistent palette.

Once we had a clear design in place, we divided the workload. Each team member was responsible for developing one page of the website based on the design they had worked on. This approach allowed us to break down the project into manageable sections and ensured that everyone had a specific area of responsibility.

We made an intentional decision to start working on the project early. By doing so, we ensured that there were sufficient time for revisions and addressing any unforeseen issues. This proactive approach also gave us room for feedback and provided time to refine our work through multiple rounds of revisions.

Development phase

For the development of the website, we relied on several key tools and technologies:

  • VSCode was the code editor of choice for all team members. This tool provided essential features like syntax highlighting, version control integration, and extensions for debugging, making coding more efficient and error-free.
  • Git was used for version control, which helped us manage and track changes to the codebase. By using Git, we were able to easily collaborate and merge our work without issues.
  • GitHub was our platform for storing and sharing code. It allowed for real-time collaboration and code reviews, it also acted as a backup for our work. The GitHub repository enabled us to easily keep track of changes, resolve issues, and manage different versions of the project.

The website itself eas built using HTML, CSS, and JavaScript. HTML provided the structure of the web pages, CSS was used to style pages and make them visually appealing, and JavaScript allowed us to add interactivity. We also gained basic exposure to working with APIs to fetch dynamic content from external sources and integrate them into websites, enhancing the overall functionality.

the development phase, we also used GitHub’s issue tracker to document and prioritize bugs, features, and other tasks. This helped keep the project organized and ensured that we stayed on track to meet deadlines.

Challenges Faced

Despite our early start, we faced challenges:

  • Learning Curve: Not all team memberswerefamiliar with tools like Git ang API integeration, requiring extra time for self-learning.
  • Time Management: Balancing the project with academic responsibilities was difficult, leading to occasional delays.
  • Team Attendances: Not everyone could attend every meeting, causing communication gaps and slow progress.

What We Learned:

  • Teamwork and collaboration
  • The importance of dividing tasks among team members for better efficiency
  • Time management skills, ensuring timely progress
  • The value of starting early and finishing tasks ahead of deadlines

Technical Skills Gained:

  • Version Control with Git and GitHub: Learned how to use Git for version control and collaborate efficiently using GitHub.
  • Web Design Process: Understood the importance of planning and designing a webpage before starting development.
  • Front-End Development: Gained proficiency in HTML, CSS, and JavaScript, and learned how to link them to create cohesive web pages.
  • API: Learned how to use JSON APIs and their role in enhancing website functionality.
  • Interactive Web Development: Building an interactive website that engages users.

Conclusion

This project was highly valuable learning experiences that strengthened both our technical skills and our ability to work as a team. The challenges we faced, whether it was mastering new tools, balancing time managemnt, or improving communicatio, it helped us grow and develop essential problem-solving skills. These lessons will be invaluable in future projects and professional endeavors, giving us a solid founadation for continued growth in web development and collaboration.
